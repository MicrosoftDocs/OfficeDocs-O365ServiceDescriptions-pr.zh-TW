# <a name="microsoft-365-government---how-to-buy"></a>Microsoft 365 政府版 - 如何購買

針對美國公共機關的獨特且不斷演進的需求，Microsoft 為美國公家機關客戶建立了特定的 Microsoft 365 政府環境。 本服務描述提供環境概觀，以及如何購買之詳細資訊。 建議您同時閱讀此服務描述及其他 [Microsoft 365 和 Office 365 服務描述](../../office-365-service-descriptions-technet-library.md)。
  
## <a name="whats-microsoft-365-government"></a>什麼是 Microsoft 365 政府版？

Microsoft 365 政府版是專為美國政府機關和承包商量身打造的一組生產力、安全性與行動性雲端軟體功能，支援控管、未分類資訊。 Microsoft 365 政府版透過符合最嚴格合規性要求的獨特環境提供，是一項適用于美國政府客戶的雲端優惠，盡可能符合 Microsoft 商業雲端企業方案的功能。
  
## <a name="whats-the-value-of-microsoft-365-government"></a>Microsoft 365 政府版的價值是什麼？

Microsoft 365 政府版可協助機構有效管理法規資料合規性要求，以便隨時隨地協助美國政府領導人和員工提高生產力，其方式為啟用行動生產力並安全地與合規性共同作業。 下表提供重要政府角色的範例。<br><br>
 
| 政府領導人可以... | 政府員工可以... | 現場人員可以... |
|:-----|:-----|:-----|:-----|:-----|
|透過改善溝通和透明度提高公開信任度  <br/> |透過更好的共同合作改善服務 <br/> | 在人員需要時透過行動存取訊號欄位人員，以改善服務品質  |
|改善可協助制定政策和預算決策之深入見解的存取權  <br/> |強化跨機構溝通。  <br/> |讓共同合作和知識分享快速找出並解決問題  <br/> |
|在辦公室或在路上，與教職員和職員保持連帶關係  <br/> |建立對行為方式的深入瞭解，協助提升員工保留率  <br/> | 在機關位置建立更強大的社群感  <br/> |
|確保符合美國政府規範的解決方案，可協助降低成本並提高品質  <br/> |||

## <a name="what-makes-microsoft-365-government-different-from-microsoft-365-for-enterprise-offerings"></a>為什麼 Microsoft 365 政府版與 Microsoft 365 企業版方案不同？

Microsoft 365 政府版採用專門設計，能解決美國政府機關在四大領域合規性要求：

**專屬社群。** 政府資料中心僅適用于政府機關或部門，或已獲授權代表美國政府保存及控管資訊的商業公司。

**已篩選的人員。** 您組織的客戶內容存取權僅限於美國員工且通過必要的背景檢查的經過篩選的 Microsoft 人員。
協力廠商稽核。 所有基礎結構是由通過認證的協力廠商稽核員稽核，該稽核員會提供安全性評定報告或聯邦機構用來發出授權以 (ATO) 。 

**內容儲存空間。** 資料會儲存于美國符合規範的基礎結構內。 下表摘要列出 Microsoft 365 政府環境與承諾。<br><br>

| 產品 | 承諾 |
|:----------|:----------------------------------|
|Microsoft 365 政府 GCC  <br/> |FedRamp High、CJIS、IRS 1075、DISA SRG L2 <br/> |
|Microsoft 365 政府 GCC High  <br/> |FedRamp High、ITAR、DFARS、DISA SRG L4 控制項 <br/> |
|Microsoft 365 政府版 DoD  <br/> |DISA SRG L5  <br/> |

有關 Microsoft 安全性與合規性詳細資訊，請流覽 [Microsoft 信任中心 ](https://www.microsoft.com/trustcenter/default.aspx)  

## <a name="microsoft-365-government-eligibility-and-validation"></a>Microsoft 365 政府版資格與驗證

Microsoft 365 政府版可供持有或處理受控制資訊之合格政府客戶及非政府組織使用。 這兩種客戶類別的資格準則在 Microsoft Government Cloud 中是一致的。

GCC 和 GCC High 方案均提供給符合 Microsoft 政府雲端資格的任何客戶使用，DoD 環境供美國商務部專屬使用。 服務可用性和價格會有所差異，而 GCC 仍然是所有未保存 FedRAMP High 或 DoD 控管的 CUI 未分類資訊 (提供) 。

合格的政府客戶由下列其中一項定義：

- 聯邦機關，定義為機關、辦公室、機關、部門或其他美國政府機構
- 定義為下列其中一種的狀態/本實體：
    - 美國任何州或的地方政府機關。
    - 客戶州的法律所建立且位於客戶州管轄及地理邊界內的任何美國郡、波羅爾、波茲、縣/市、市鎮、特殊目的地區或其他類似的政府地點類型
    - 哥倫比亞地區、波多黎各的維爾京群島、美屬維爾京群島、美屬維爾京群島及北利亞納群島
- 國家實體，定義為聯邦認可且符合向美國商務部取得資金和服務資格之實體，其狀態為印度使用者，或位於烏克蘭的印地安達或烏克蘭地區原生公司

具備下列資料類型，並且可提供符合 Microsoft 政府雲端驗證程式資格之所列證明的非政府組織：

這是具有受法規保護之資料的商業私人實體。 接受政府資料類型包括：
- ITAR (國際) 
- 受控的未分類資訊 (CUI) 
- 美國商務部 (DOD) UCNI (控管的未分類) 
- 能源部門 (DoE) UCNI
- CJI (犯罪資訊) 
- 美國商務部影響層級資料
- 需要 Microsoft 365 政府版的其他資料類型

國際商業實體可能符合資格，但受規範的資料可能需要透過其美國子公司購買服務。

如要存取 Microsoft 365 政府版方案，必須取得上述其中一個群組的成員資格證明。
